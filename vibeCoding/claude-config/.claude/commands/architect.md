# /architect - 架构设计

**触发词**: 架构、设计、技术选型、方案对比

## 工作流

```
加载AR角色 → 深度分析 → 方案设计 → 寸止确认
```

## 执行步骤

### 1. 加载角色
```
promptx.switch("AR")
```

### 2. 深度分析
```
sequential-thinking 推演:
- 系统边界
- 技术约束
- 可行方案
```

### 3. Linus审查
- [ ] 数据结构最简？
- [ ] 命名准确？
- [ ] 是否过度设计？
- [ ] 向后兼容？

### 4. 方案对比
```markdown
| 维度 | 方案A | 方案B |
|------|-------|-------|
| 复杂度 | 低 | 中 |
| 性能 | 中 | 高 |
| 成本 | 1周 | 2周 |
```

### 5. 寸止确认
```
多方案 → 寸止让用户选择
禁止自作主张
```

## 注意

此命令专注于设计咨询，**不生成代码**。

实现请用 /dev 或 /quick
