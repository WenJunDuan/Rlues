# /review - 代码审查

**触发词**: 审查、review、检查代码

## 工作流

```
sou搜索 → 代码分析 → 问题分级 → 寸止报告
```

## 审查维度

| 维度 | 权重 | 检查点 |
|------|------|--------|
| 功能正确 | 40% | 逻辑、边缘、错误处理 |
| 代码质量 | 30% | 可读性、命名、结构 |
| 安全性 | 20% | 输入验证、注入防护 |
| 性能 | 10% | 效率、资源使用 |

## 问题分级

| 级别 | 标识 | 行动 |
|------|------|------|
| Critical | 🔴 | 必须修复 |
| Major | 🟠 | 强烈建议 |
| Minor | 🟡 | 建议修改 |
| Info | 🟢 | 参考建议 |

## 检查清单

- [ ] TypeScript 无 `any`
- [ ] 函数 <50行
- [ ] 组件 <200行
- [ ] 完整错误处理
- [ ] 输入验证
- [ ] 无注入风险

## 输出格式

```markdown
## 代码审查报告

**文件**: {path}
**评分**: {X}/100

### 🔴 必须修复
1. [行号] 问题描述 → 建议

### 🟠 建议修改
1. ...

### ✅ 优点
1. ...
```

## 寸止报告

审查完成后通过寸止报告结果，等待用户确认
