# P.A.C.E. 复杂度路由 (Codex)

## 路由判定

| 信号 | Path A | Path B | Path C | Path D |
|:---|:---|:---|:---|:---|
| 文件数 | 1-2 | 3-10 | 10+ | 20+ |
| 预估时间 | ≤30 min | 6-12h | 1-3 周 | 3+ 周 |
| 关键词 | 修/改/加/删 | 做/实现/开发 | 重构/迁移/系统 | 架构/平台/全栈 |
| 描述长度 | ≤50 字 | 50-200 字 | 200+ 字 | 200+ 字+多模块 |
| 需求数 | 1 | 2-5 | 5-15 | 15+ |

不确定时: 问用户, cunzhi 确认。

## 分级加载

### Path A — 敏捷
```
加载: AGENTS.md
RIPER: R(轻) → E → T(轻) → V(cunzhi) → Done
```

### Path B — 协作
```
加载: AGENTS.md + workflows + 相关 skills
RIPER: R₀b → R → D → P → E → T → V → Done
```

### Path C — 系统
```
加载: 全量
额外: /collab 启动协作 + 并发 shell
```

### Path D — 长期
```
同 Path C, 按周拆分里程碑
```
