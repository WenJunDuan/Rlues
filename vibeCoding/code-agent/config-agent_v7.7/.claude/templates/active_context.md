# Active Context 模板

## 使用说明
此文件记录当前会话的上下文信息，用于：
- 会话恢复
- 上下文传递
- 状态追踪

## 更新时机
- 会话开始
- 阶段转换
- 任务完成
- 重要决策

---

## 模板

```markdown
# 当前上下文

## 基本信息
- **项目**: [项目名称]
- **更新时间**: [timestamp]
- **AI 引擎**: claude

---

## 当前任务
### 描述
[任务的详细描述]

### 状态
- **路径**: Path B
- **阶段**: Execute
- **进度**: 3/5 任务完成

### 涉及文件
- `src/components/Settings.tsx` - 主要修改
- `src/api/settings.ts` - API 调用
- `src/types/settings.ts` - 类型定义

---

## 关键上下文
### 需求理解
[对需求的理解摘要]

### 技术决策
| 决策 | 选择 | 原因 |
|:---|:---|:---|
| 状态管理 | Zustand | 轻量、简单 |
| API 风格 | RESTful | 团队熟悉 |

### 注意事项
- [注意事项1]
- [注意事项2]

---

## 最近操作
| 时间 | 操作 | 结果 |
|:---|:---|:---|
| 10:30 | 完成 Settings.tsx | ✓ |
| 10:15 | 完成 types.ts | ✓ |
| 10:00 | 开始 Execute 阶段 | - |

---

## 待解决问题
- [ ] [问题1]
- [ ] [问题2]

---

## 下一步
[下一步计划]
```

---

## 字段说明

### 基本信息
```yaml
项目: 当前工作的项目名称
更新时间: 最后更新时间戳
AI引擎: 当前使用的 AI 引擎
```

### 当前任务
```yaml
描述: 任务的完整描述
状态: 路径、阶段、进度
涉及文件: 本次任务涉及的文件列表
```

### 关键上下文
```yaml
需求理解: Research 阶段的需求分析结果
技术决策: Innovate 阶段的决策记录
注意事项: 开发过程中需要注意的点
```

### 最近操作
```yaml
格式: 时间 | 操作 | 结果
用途: 追踪操作历史，便于恢复
```

---

## 与其他文件的关系
```yaml
session.lock: 会话元数据 (ID、状态)
active_context.md: 详细上下文 (任务、决策)
kanban.md: 任务状态
TODO.md: 完整任务列表

层次:
  session.lock (简) → active_context.md (详) → TODO.md/kanban.md (执行)
```
