# P.A.C.E. 智能路由

## 评估维度

| 维度 | 低(1-3) | 中(4-6) | 高(7-10) |
|------|---------|---------|----------|
| 文件数 | 1个 | 2-10个 | >10个 |
| 代码量 | <30行 | 30-300行 | >300行 |
| 时间 | <30分钟 | 30分-4小时 | >4小时 |
| 架构影响 | 无 | 局部 | 全局 |

## 路径选择

### ⚡ Path A — 闪电模式

**条件**: 单文件 / <30行 / 纯修复

**流程**: `R1 → E → R2`

**特征**:
- 静默执行
- 只展示结果
- 仅R2验收寸止
- 不召开会议

### 🤝 Path B — 协作模式

**条件**: 2-10文件 / 新功能 / 局部重构

**流程**: `R1 → I → P → E → R2`

**特征**:
- 召开技术设计会议
- 任务规划会议
- I确认 + R2验收

**参与角色**: AR + LD + SA

### 🏗️ Path C — 系统模式

**条件**: >10文件 / 架构变更 / 从0到1

**流程**: `R1 → I → P → E(迭代) → R2`

**特征**:
- 完整多角色协作
- 分阶段交付
- 每阶段寸止确认

**参与角色**: PM + PDM + AR + LD + QE + SA

## 决策流程

```
收到任务
    ↓
评估四个维度
    ↓
计算综合分数
    ↓
┌─────────────────────────────────┐
│ 1-3分 → Path A                  │
│ 4-6分 → Path B                  │
│ 7-10分 → Path C                 │
└─────────────────────────────────┘
    ↓
加载对应工作流和角色
```

## 用户覆盖

用户可显式指定：
- "用完整RIPER" → 强制Path C
- "快速修复" → 强制Path A
- "简化流程" → 强制Path B
