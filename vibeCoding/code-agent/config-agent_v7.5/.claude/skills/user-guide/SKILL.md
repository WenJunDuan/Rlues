## 1. 快速开始

### 1.1 第一次使用

**你说**：
```
我想做一个 [你的想法]
```

**AI 会**：
1. 问你几个问题（确认需求）
2. 给你一个计划（任务列表 + 时间预估）
3. 等你确认

**你说**：
```
确认
```

**然后**：AI 开始工作，你可以随时查看进度。

### 1.2 示例对话

```
👤 你：我想做一个待办事项应用

🤖 AI：好的！让我确认几个问题：
    1. 需要用户登录吗？
    2. 要支持手机端吗？
    3. 需要提醒功能吗？

👤 你：要登录，先做网页版，暂时不要提醒

🤖 AI：明白了！我来规划任务：

    📋 任务计划：
    1. 项目初始化 (10分钟)
    2. 数据库设计 (20分钟)
    3. 用户登录功能 (1小时)
    4. 待办增删改查 (1小时)
    5. 页面界面 (1.5小时)
    
    总计：约 4 小时
    
    📋 [等待确认]
    请回复：确认 / 修改 / 取消

👤 你：确认

🤖 AI：开始执行！你可以随时说"进度"查看当前状态。
```

## 2. 日常使用

### 2.1 你可以说的话

| 你说 | AI 做什么 |
|:----|:---------|
| `进度` / `怎么样了` | 显示当前进度看板 |
| `继续` | 恢复上次的工作 |
| `暂停` | 保存当前状态，下次继续 |
| `改一下 T-003` | 修改某个任务的要求 |
| `用 codex 做这个` | 指定用某个 AI 执行 |
| `这个任务不要了` | 取消某个任务 |
| `加一个功能` | 在现有计划中添加新任务 |

### 2.2 AI 会问你的时机

AI 不会自作主张，以下情况会等你确认：

| 时机 | AI 会说 |
|:----|:-------|
| 计划做好后 | "计划已完成，请确认" |
| 遇到多个方案 | "有两个方案，你选哪个？" |
| 一个阶段完成 | "Phase 1 完成，继续吗？" |
| 遇到问题 | "遇到一个问题，需要你决定..." |
| 全部完成 | "项目完成，请验收" |

### 2.3 指定 AI 工具

默认情况下，AI 自动选择最合适的工具。你也可以指定：

```
用 codex 来做这个登录功能
让 gemini 处理这个文件
这个用 claude 来做
```

## 4. 常见问题

### Q: 我不懂这些技术名词怎么办？

**A**: 不用懂！AI 会用你能理解的方式解释。如果看到不懂的，直接问：
```
"什么是 SQLite？"
"为什么要用这个？"
```

### Q: AI 做的不是我想要的怎么办？

**A**: 直接说：
```
"这个不对，我想要的是..."
"改一下，应该是..."
"回退这个修改"
```

### Q: 我想中途改需求怎么办？

**A**: 随时可以说：
```
"等一下，我想加一个功能..."
"这个功能不要了"
"改一下 T-003，我想让它..."
```

### Q: 如何选择用哪个 AI？

**A**: 通常你不需要选，系统会自动选。但如果你想指定：
```
"用 codex 做这个"    ← 快速执行简单任务
"用 claude 做这个"   ← 需要深度思考
"用 gemini 做这个"   ← 处理很长的文件
```

### Q: AI 做错了怎么办？

**A**: 
```
"这里有 bug"         ← AI 会自动修复
"撤销这次修改"       ← 回退到之前的版本
"重新做 T-003"       ← 重新执行某个任务
```

### Q: 项目做完后怎么办？

**A**: AI 会给你：
- 完整的代码
- 部署说明
- 使用文档

你可以说：
```
"怎么运行这个项目？"
"怎么部署到服务器？"
"给我一个说明文档"
```

## 5. 进阶用法

### 5.1 并行执行（更快）

当有多个互不相关的任务时，可以说：
```
"T-004 和 T-005 可以同时做"
```
AI 会并行执行，节省时间。

### 5.2 分阶段验收

大项目建议分阶段验收：
```
"先做完登录功能给我看看"
"这个阶段没问题，继续下一个"
```

### 5.3 保存和恢复

```
"暂停一下，明天继续"    ← 保存状态
（第二天）
"继续昨天的项目"        ← 恢复状态
```
