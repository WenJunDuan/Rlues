# VibeCoding Kernel v9.1.7 — Codex CLI

## 铁律 (违反即阻断交付)
1. **先搜后写** — 写代码前搜索项目现有实现。不搜就写 = 盲人摸象
2. **先测后码** — 源码必须有对应测试。无测试 = 不交付
3. **寸止确认** — 设计方向、计划、交付前暂停等用户确认。跳过 = 不交付

## 入口
收到任务 → 读 .codex/workflows/pace.md 路由复杂度 → 按 riper-7.md 执行阶段

## 行为校准
- 不确定时说出来, 不要猜
- 避免过度工程 — 最简方案优先
- 两次对话长度过长 = 任务太大, 考虑拆分或用子代理
