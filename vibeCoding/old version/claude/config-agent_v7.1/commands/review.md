# /review - 代码审查

**触发词**: 审查、review、检查代码

## 核心理念

> **事实为本**: 基于事实评审，不基于偏好

## 工作流

```
sou搜索 → Linus审查 → 问题分级 → 寸止报告
```

## Linus 审查清单

### 设计层面
```markdown
- [ ] **Data First**: 数据结构是最简的吗？
- [ ] **Naming**: 命名准确反映本质？
- [ ] **Simplicity**: 是否过度设计？能删除什么？
- [ ] **Taste**: 这段代码有"好味道"吗？
```

### 代码层面
```markdown
- [ ] TypeScript 无 `any`
- [ ] 函数 <50行
- [ ] 组件 <200行
- [ ] 完整的错误处理
- [ ] 输入验证
```

### 安全层面
```markdown
- [ ] 输入验证
- [ ] 无注入风险
- [ ] 敏感数据处理
```

## 问题分级

| 级别 | 标识 | 行动 |
|------|------|------|
| Critical | 🔴 | 必须修复 |
| Major | 🟠 | 强烈建议 |
| Minor | 🟡 | 建议修改 |

```markdown
## 代码审查报告

**文件**: {path}

### Linus评审

1. ...

1. ...
```
