# Reflexion Skill: 自我审查

## 你的目标

在每个 Task 完成后、交给外部 Review 之前，对自己的工作做一次诚实的自我检查。
发现问题立即修复，不留给 Review 阶段。

---

## 审查流程

完成一个 Task 的代码 + 测试后，回答以下 3 个问题:

### 问题 1: 完整性
- 验收标准是否每一条都实现了? 逐条对照 feature_list.json 的 acceptance_criteria。
- 有没有 "差不多了" 的地方? 差不多 = 没完成。

### 问题 2: 边界和异常
- 空值/null 怎么处理?
- 并发访问安全吗?
- 输入超出预期范围怎么办?
- 网络请求失败怎么处理?
- 错误消息对用户友好吗?

### 问题 3: 简洁性
- 有没有不必要的代码? (未使用的 import、注释掉的代码、调试日志)
- 有没有更简单的实现方式?
- 代码是否符合 rules/code-standards.md 的 P0 和 P1?

---

## 审查结果

**发现问题:** 立即修复 → 重新运行测试 → 确认通过 → 再做一轮 reflexion。
**无问题:** 在心里确认 "已 reflexion, 无问题" → 继续下一个 Task 或进入 T 阶段。

---

## Gotchas

- ❌ reflexion 走过场，"看起来没问题" → ✅ 逐条对照验收标准，每条都要检查
- ❌ 发现问题但想 "Review 阶段会捕获" → ✅ 现在修，不留后患
- ❌ 跳过 reflexion 直接进入下一个 Task → ✅ 这违反铁律 #4
