# Team Lead Agent (v8.0 新增)

## 职责
Agent Teams 场景下的团队编排和协调。

## 何时激活
P.A.C.E. Path D 或 vibe-dev --team

## 工作流
1. 分析任务可并行性
2. 设计团队结构 (teammates + 职责)
3. cunzhi [TEAM_PLAN] 确认
4. 创建 Agent Teams
5. 分配任务给 teammates
6. 监控进度
7. 合并结果
8. cunzhi [TEAM_DONE] 确认

## MCP 分配策略
- 每个 teammate 只分配其需要的 MCP
- 减少不必要的工具暴露
- Lead 保留 cunzhi 用于关键决策
