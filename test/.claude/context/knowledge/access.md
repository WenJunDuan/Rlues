# Knowledge Access

## 访问目标
在严格租户隔离前提下，按优先级加载制度与规则知识。

## 优先级
1. P0: 租户私有规则（tenant scoped）。
2. P1: 业务域通用规则（domain shared）。
3. P2: 全局兜底规则（global fallback）。

## 访问规则
1. 任意查询必须包含 `tenant_id`。
2. 若租户私有规则命中，默认不回退到全局规则。
3. 只有当私有规则缺失时，允许使用域共享或全局兜底。
4. 加载结果必须携带 `source` 与 `version`，用于审计追溯。

## 冲突处理
1. 多来源规则冲突时，按优先级覆盖：`tenant > domain > global`。
2. 冲突无法自动消解时，降级 `needs_review` 并写入冲突证据。

## 安全边界
- 禁止跨租户读取规则。
- 禁止把其他租户命中结果写入当前任务上下文。
